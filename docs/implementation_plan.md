# 宿泊行事の栞（しおり）作成アプリケーション 実装計画

修学旅行や宿泊行事向けの「栞（しおり）」を簡単に作成できるWebアプリケーションを構築します。ご要望の通り、日時や行き先などを別々のページで入力でき、今後拡張できる柔軟な設計とします。

## 背景と目的
- **場所**: `Gyoji-shiori` フォルダ
- **技術スタック**: React + Vite + React Router + Vanilla CSS
  - 複雑な状態管理やルーティングをスムーズに行うため、Reactを採用します。
  - TailwindCSS等のフレームワークは使用せず、柔軟でモダンな美しいUIをVanilla CSSで構築します。
- **機能**: 各項目を別々のページで入力・編集でき、最後に全体をプレビューできる仕組みを作ります。入力データはブラウザ（ローカルストレージ）に自動保存され、リロードしても消えないようにします。

## User Review Required

> [!IMPORTANT]
> 以下の点について、実装を進める前にご確認ください。
> 1. **「もちもの」の重複**: ご要望のリスト内で、4番と8番に「もちもの」が2回登場しています。今回は1つにまとめて実装する想定でよろしいでしょうか？（もし「個人用」と「班用」などで分ける場合はお知らせください）
> 2. **最終出力のイメージ**: 入力した内容を最終的に1つの「しおり」として閲覧・印刷できる「プレビューページ（完成版ページ）」を用意する方針でよろしいでしょうか？

## Proposed Changes

### 1. プロジェクトのセットアップ
`Gyoji-shiori` フォルダ内に、Viteを使用してReactプロジェクトをセットアップします。
- **依存関係の追加**: `react-router-dom` (画面遷移用), `lucide-react` (アイコン用)

### 2. ルーティングと状態管理の構築
全体のデータを管理するContextを作成し、各ページでデータを入力・参照できるようにします。

#### [NEW] `src/context/ShioriContext.jsx`
入力された栞のデータ（日時、行き先、目標など）を一元管理し、ローカルストレージに保存するコンテキスト。

#### [NEW] `src/App.jsx` & `src/main.jsx`
React Routerの設定を行い、サイドバーまたはナビゲーションバーから各ページへ遷移できるようにします。

### 3. 各入力ページの作成
ご要望の項目ごとの入力用コンポーネントを作成します。

#### [NEW] `src/pages/Page1_DateTime.jsx` (日時)
#### [NEW] `src/pages/Page2_Destination.jsx` (行き先)
#### [NEW] `src/pages/Page3_Goals.jsx` (目標)
#### [NEW] `src/pages/Page4_Belongings.jsx` (もちもの)
#### [NEW] `src/pages/Page5_Schedule.jsx` (スケジュール)
#### [NEW] `src/pages/Page6_Seats.jsx` (バス・電車の座席 - 車両や席の配置を選べるUI)
#### [NEW] `src/pages/Page7_Room.jsx` (泊まる部屋 - 部屋割りと同室メンバー)
#### [NEW] `src/pages/Page9_PocketMoney.jsx` (お小遣い帳 - 予算と支出のリスト)
#### [NEW] `src/pages/Page10_Roles.jsx` (かかり - 班長、保健係などの役割分担)
#### [NEW] `src/pages/Page11to15_Custom.jsx` (拡張ページ1〜5 - 見出しと内容を自由に入力できる汎用ページ)

### 4. プレビュー画面の作成
#### [NEW] `src/pages/Preview.jsx`
入力したすべてのデータを「しおり」の形式で一覧表示し、印刷（PDF化）しやすいレイアウトで表示するページ。

### 5. デザインシステム (Vanilla CSS)
#### [MODIFY] `src/index.css` / `src/App.css`
モダンで直感的なUIを提供するため、以下のデザインを適用します。
- 丸みのあるカードデザイン、ガラスモーフィズム（すりガラス効果）
- ユーザーの操作に対するスムーズなマイクロアニメーション（ホバー効果など）
- 読みやすいタイポグラフィ（Google Fontsのインポート）
- 美しいグラデーション背景と洗練されたカラーパレット

## Verification Plan

### 自動・手動テスト
1. `npm run dev` でローカルサーバーを起動し、ブラウザで画面を確認します。
2. ナビゲーションを使って各ページ（1〜15）に正しく遷移できるか確認します。
3. 各ページでデータを入力後、ページを移動して戻ってきてもデータが保持されているか（ローカルストレージの動作）を確認します。
4. 全体のデザインがモダンでリッチな見た目になっているか（Vanilla CSSの適用確認）を検証します。
5. 座席選択ページで「バス」「電車」の切り替えと入力が正しく機能するか確認します。
