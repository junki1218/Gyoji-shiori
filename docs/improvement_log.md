## サイクル 1（2026-06-30）

### 発見したバグ・問題点

| No. | 重要度 | 場所 | 問題 |
|-----|--------|------|------|
| B-01 | 🔴 致命的 | `PageCustom.jsx` | フリーページに `content` フィールドを入力するUIが存在しない。Preview/Print は `p.content` でフィルタリングするため、フリーページがしおりに一切出力されない |
| B-02 | 🟠 高 | `ShioriContext.jsx` | localStorage からの復元時に `pocketMoney` が浅いマージのみ。将来フィールド追加時にデフォルト値が適用されない |
| B-03 | 🟡 中 | `Page4_Belongings.jsx`, `Page7_Room.jsx`, `Page9_PocketMoney.jsx` | `onKeyPress` は React 17以降で非推奨。`onKeyDown` に変更が必要 |
| B-04 | 🟡 中 | `Page6_Seats.jsx` | 列数・行数を縮小しても範囲外のアサインメントが `assignments` オブジェクトに残留する |
| B-05 | 🟡 中 | `App.jsx` | サイドバーのアクティブリンクがスクロール外に出ても自動スクロールしない |
| B-06 | 🟢 低 | `PageCustom.jsx` | コメント削除・完了ボタンに `.btn` クラスが未付与でスタイルなし |

### 修正プラン

**B-01（致命的）**: `PageCustom.jsx` にテキストエリアを追加し、`content` フィールドを編集可能にする。画像エリアの下にテキスト入力欄を設置。

**B-02（高）**: `ShioriContext.jsx` の初期化部分で `pocketMoney` も明示的にマージする。

**B-03（中）**: 全該当箇所の `onKeyPress` を `onKeyDown` に変更し、`e.key === 'Enter'` でチェック。

**B-04（中）**: `Page6_Seats.jsx` の `handleDimensionChange` で変更後の行/列数に基づいて `assignments` から範囲外エントリを削除する。

**B-05（中）**: `App.jsx` の `Sidebar` で `useEffect` + `scrollIntoView` を使ってアクティブリンクを可視領域に自動スクロールする。

**B-06（低）**: コメント操作ボタンに `btn btn-icon` / `btn btn-primary` スタイルを付与する。

## サイクル 2（2026-07-01）— PDFレポートに基づくアクセシビリティ改善

> 参照: 「知的障害児向けWebアプリ改善提案」PDFレポート（WCAG 2.0 準拠・UDデザイン原則）

### 改善案 10 項目

| No. | 分類 | 改善内容 | 根拠（PDFより） |
|-----|------|---------|--------------|
| I-01 | 操作性 | **持ち物リストの全行タップ化・大型UI** — 行全体（min 64px）をタップでチェック | WCAG 2.5.5「ターゲットサイズ 48×48dp 以上」 |
| I-02 | フィードバック | **チェック完了アニメーション** — パックすると緑フラッシュ＋✓アイコンに変化 | 「即時ポジティブフィードバック」はモチベーション維持に効果的 |
| I-03 | 視認性 | **持ち物完了率プログレスバー** — 「X / Y 個 準備完了」をカラー変化バーで表示 | スモールステップ達成感の可視化による遂行能力支援 |
| I-04 | 感情表現 | **感情スタンプ（メモページ）** — 5段階の絵文字スタンプ（😊😄😐😪😢）を選択して保存 | 文字入力が困難な子どもへのマルチモーダル入力手段 |
| I-05 | 視認性 | **Now/Next カード（スケジュール上部）** — 「今いる活動」と「次の活動」を大きく表示 | 「Now/Nextカード」は認知負荷を下げ現在状況把握を容易に |
| I-06 | 視認性 | **スケジュール今日の日付を強調** — 当日の日付バッジを黄色で目立たせる | 時間的見通しの提供による不安軽減 |
| I-07 | 感知可能 | **テキスト読み上げ（TTS）浮きボタン** — Web Speech API でページ内容を日本語音声で読み上げ | WCAG 1.1「テキストの代替」・文字認識困難者の支援 |
| I-08 | 視認性 | **高コントラスト改善** — form ラベル・muted テキストの透明度を 60%→75% 以上へ引き上げ | WCAG 1.4.3「コントラスト比 4.5:1 以上」 |
| I-09 | 操作性 | **ナビゲーションリンク大型化** — `min-height: 48px` でタッチターゲット保証 | WCAG 2.5.5 / Google Material Design 48dp ガイドライン |
| I-10 | 視認性 | **ページ進捗インジケーター** — PageNav に「3 / 15」を表示 | ステップ数の可視化によるページ見通し支援 |

### 実装プラン

- **I-01〜I-03**: `Page4_Belongings.jsx` に `packed` フィールドを追加。行全体を `onClick` で包み、完了アニメーションは `@keyframes packFlash`
- **I-04**: `ShioriContext.jsx` に `memoEmotion: ''` を追加。`Page8_Memo.jsx` に感情スタンプ UI を実装
- **I-05〜I-06**: `Page5_Schedule.jsx` に `NowNextCard` コンポーネントを追加
- **I-07**: `App.jsx` に `TTSController` コンポーネント（固定ボタン）を追加
- **I-08〜I-10**: `index.css` でコントラスト値・min-height・進捗表示を追加

---

### 実装記録

#### I-01 持ち物全行タップ化 ✅
- `belonging-item` が大型タッチターゲット（min 64px）として機能
- `packed` フィールドで「詰めた」状態を管理（`isEssential` は「必須」マーカーに分離）
- キーボード操作対応（Space/Enter でもトグル可能）

#### I-02 チェック完了アニメーション ✅
- `recentlyPacked` ステートで直後 900ms 間 `.belonging-flash` クラスを付与
- `@keyframes packFlash` で緑フラッシュ＋スケール拡大を実現

#### I-03 持ち物完了率プログレスバー ✅
- カード上部に X/Y + カラー変化バーを表示
- 0-80%: 黄色、80-99%: オレンジ、100%: 緑＋「🎒 準備完了！」メッセージ

#### I-04 感情スタンプ ✅
- `ShioriContext` に `memoEmotion: ''` を追加（既存セーブデータと後方互換）
- 5 種類の絵文字スタンプ（押すと選択/解除トグル）
- 選択中スタンプは大きく表示、黄色いハイライト枠

#### I-05 Now/Next カード ✅
- 今日の予定がある場合のみスケジュールページ上部に表示
- 現在時刻比較で「NOW」（現在の活動）と「NEXT」（次の活動）を強調カード表示

#### I-06 スケジュール今日強調 ✅
- 今日の日付バッジに `today-date-badge` クラスを追加、常時 yellow 表示

#### I-07 TTS 読み上げボタン ✅
- `window.speechSynthesis` 未サポート環境では非表示
- `.glass-panel` 内テキストを `ja-JP` で読み上げ（最大 2000 文字）
- ページ遷移時に自動停止、読み上げ中は脈動アニメーション

#### I-08 高コントラスト改善 ✅
- `.form-group label`: 透明度 60% → 78% へ引き上げ（〜6:1 コントラスト比）
- `.text-muted`: 60% → 68% へ引き上げ

#### I-09 ナビゲーションリンク大型化 ✅
- `.nav-link` に `min-height: 48px` を追加

#### I-10 ページ進捗インジケーター ✅
- PageNav の中央に「3 / 15」表示を追加（`.page-nav-progress`）
